# 部署到 GitHub Pages

## 費用：$0

**公開 repo 的 GitHub Pages 完全免費** —— 沒有流量費、沒有時間限制。
（只有「私有 repo 也要用 Pages」才需要付費方案，你不需要。）

---

## 我已經幫你做好的

`D:\train_ticket_buyer_web` 資料夾已經是一個**完整的 git 倉庫**，commit 都建好了，工作區乾淨。你只差「推上去」這一步。

也加了兩個檔案：

- `.gitignore` — 排除 `__pycache__` 等雜物
- `.nojekyll` — 讓 GitHub Pages 直接吐檔案，不跑 Jekyll。**這個有必要**：Jekyll 預設會忽略底線開頭的檔案，而我們的產生器就叫 `_build.py` / `_content_*.py`

---

## 為什麼要你自己推

推送需要你的 GitHub 帳號認證（token 或 SSH 金鑰）。**我不會經手你的帳號憑證** —— 這類東西不應該交給任何第三方，包括我。所以最後這一步請你自己做。

---

## 步驟

### 1. 在 GitHub 建一個空的 repo

到 https://github.com/new

- Repository name：`alishan-booking`（或你喜歡的名字）
- 選 **Public**（Pages 免費的前提）
- **不要**勾選 "Add a README"、.gitignore、license —— 我們已經有了，勾了會衝突

建好後 GitHub 會給你一個網址，長這樣：
```
https://github.com/你的帳號/alishan-booking.git
```

### 2. 推上去

在你電腦的 `D:\train_ticket_buyer_web` 資料夾開啟終端機（PowerShell 或 Git Bash），執行：

```bash
git remote add origin https://github.com/你的帳號/alishan-booking.git
git push -u origin main
```

會要求登入。現在 GitHub 不接受密碼，要用 **Personal Access Token**：
建立位置 → https://github.com/settings/tokens → Generate new token (classic) → 勾 `repo` 權限。
推送時帳號填你的 GitHub 帳號、密碼欄貼那個 token。

> 如果你電腦裝了 GitHub Desktop，直接用它推更簡單，不用處理 token。

### 3. 開啟 Pages

repo 頁面 → **Settings** → 左側 **Pages**

- Source：`Deploy from a branch`
- Branch：`main`，資料夾選 `/ (root)`
- 按 **Save**

等 1–2 分鐘，網址會出現在同一頁上方。

### 4. 你會得到的網址

```
首頁          https://你的帳號.github.io/alishan-booking/
隱私權政策     https://你的帳號.github.io/alishan-booking/privacy.html
English       https://你的帳號.github.io/alishan-booking/en/
日本語         https://你的帳號.github.io/alishan-booking/ja/
```

**`privacy.html` 那個網址就是 Chrome 開發者後台要填的隱私權政策網址。**

---

## ⚠️ 一個隱私提醒

我用的 commit 作者是 `noreply@example.com`（佔位用）。如果你之後用自己的帳號 commit，**你的 email 會出現在公開 repo 的提交紀錄裡**。

想避免的話，在 repo 資料夾執行：

```bash
git config user.email "你的帳號ID+你的帳號@users.noreply.github.com"
```

這個 noreply 位址可以在 https://github.com/settings/emails 找到。

---

## 之後要改內容

1. 改 `_content_zh.py` / `_content_en.py` / `_content_ja.py`
2. 跑 `python3 _gen.py` 重新產生 HTML
3. `git add -A && git commit -m "更新內容" && git push`

推上去後 Pages 會自動重新部署，大約 1 分鐘生效。

**不要直接改 HTML** —— 下次跑產生器會被覆蓋，而且三語言容易走鐘。
